# 회의내용

1. 기존의 프로젝트의 계획
   - 프론트엔트 (챗봇 앱) → google Dialogflow → 서버 (웹 훅 서버 = dialogFlow에서 보내는 데이터를 받아서 crawling을 진행하고, 진행한 결과를 재가공하여 다시 보내주는 서버) 

근데 이때 구현서버에서 크롤링을 진행할 때 문제가 발생한다.

**문제1**. 홈페이지에 들어가서 내부에서 크롤링을 할 때 종목이 너무 많아서 한, 두 개의 검색어로는 제대로된 크롤링이 불가능하다. 그렇다고 기존의 국민비서 처럼 큰 주제를 미리 선택할 수 있듯이 만들어주고 하는 방법으로 구현을 하면 국민비서와 다를 것이 없을 것 같다.

 => 3개월 정도의 연구를 통해서 할 가치가 있을지에 대한 의문이 든다.. 차라리 주제를 좀 더 자유롭게 바꾸는 것은 어떠한가 

**문제2.** 지자체 사이트에서 검색을 할 시 비슷한 키워드여도 다른 결과가 나올 수 있고, 결과의 차이가 발생할 수 있다.
